## üìã WHERE CREDENTIALS ARE STORED

### **Option 1: .env File (Primary)**

**Location**: `.env` in repository root

**Required Variables**:
```env
HOSTINGER_HOST=157.173.214.121
HOSTINGER_USER=your_username
HOSTINGER_PASS=your_password
HOSTINGER_PORT=65002
```

**Alternative Variable Names** (also supported):
- `SSH_HOST` / `HOST` (instead of `HOSTINGER_HOST`)
- `SSH_USER` / `USERNAME` (instead of `HOSTINGER_USER`)
- `SSH_PASS` / `PASSWORD` (instead of `HOSTINGER_PASS`)
- `SSH_PORT` / `PORT` (instead of `HOSTINGER_PORT`)

### **Option 2: sites.json (Site-Specific)**

**Location**: `.deploy_credentials/sites.json`

**Structure**:

  }

## üîß HOW TO GET SFTP CREDENTIALS FROM HOSTINGER

### **Step 1: Log into Hostinger Control Panel**

1. Go to: **https://hpanel.hostinger.com/**
2. Log in with your Hostinger account credentials

### **Step 2: Find SFTP Credentials**

**Method A: FTP Accounts (Recommended)**
1. Navigate to: **Files** ‚Üí **FTP Accounts**
2. Find your domain (e.g., `freerideinvestor.com`)
3. You'll see:
   - **FTP Username** (may be different from your email)
   - **FTP Password** (click "Show" or "Reset" if needed)
   - **FTP Host** (usually an IP like `157.173.214.121`)
   - **FTP Port** (should be `65002` for SFTP)

**Method B: File Manager**
1. Navigate to: **Files** ‚Üí **File Manager**
2. Look for SFTP/FTP connection details in the interface

**Method C: Hosting Settings**
1. Navigate to: **Hosting** ‚Üí **Manage** ‚Üí **Your Domain**
2. Look for **FTP/SFTP** section
3. Find connection details

### **Step 3: Important Notes**

- **Username Format**: The SFTP username might be different from your Hostinger login email. Common formats:
  - `cpanel_username` (cPanel username)
  - `u1234567` (numeric prefix)
  - `username@domain.com` (email format - less common)
  - `domain_username` (domain-based)

- **Password**: If you don't know the password, you can reset it in the FTP Accounts section. **Save it immediately** - you won't see it again!

- **Port**: Hostinger SFTP uses port **65002** (not the standard port 22)

## üîç TROUBLESHOOTING

### **If Connection Fails**:

1. **Check .env file exists**: Make sure `.env` is in repository root
2. **Verify variable names**: Use `HOSTINGER_HOST`, `HOSTINGER_USER`, `HOSTINGER_PASS`, `HOSTINGER_PORT`
3. **Check username format**: Try different username formats (see Step 3 above)
4. **Reset password**: Reset password in Hostinger and update `.env`
5. **Verify port**: Make sure port is `65002` for Hostinger SFTP

### **If SFTP Not Available**:

- Use FTP instead (port 21, different protocol)
- Or use WordPress Admin automation method
- Or use Hostinger File Manager (web-based)
